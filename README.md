# 程序员模拟器 - 肉鸽游戏

一款以程序员视角体验中国互联网发展历程的肉鸽类模拟游戏。

## 游戏简介

一觉醒来你回到了2000年，决定以程序员的身份重新开始人生。从C级独立作品做起，经历季度任务和Boss战，最终挑战风靡全国的A级产品或世界级的S级产品。

## 核心特色

### 🎮 肉鸽+模拟玩法
- **随机产品抽取**：根据眼界值从30+历史产品中随机选择
- **动态方案生成**：每个季度任务根据产品类型动态生成不同的开发方案
- **临时升级系统**：Boss战成功后获得临时增益，持续到本局结束
- **永久升级系统**：使用晶核永久提升属性，为下一局做准备

### 📊 数值体系
- **眼界 (初始50)**：决定下一个项目的品级，眼界越高能接触到越好的产品
- **运气 (初始50)**：影响项目发布后的市场反响和评级
- **体力 (初始50)**：开发过程中消耗，归零则项目失败

### 🎯 游戏流程
1. **3个季度任务**：每个季度提供3种开发方案
   - 快速方案：低体力消耗，低质量影响，快速完成
   - 平衡方案：中等消耗，标准质量
   - 精益方案：高体力消耗，高质量影响，进度慢
   
2. **1个Boss战**：整合前3个季度成果，发布最终产品
   - 需要完成度≥80%才算成功
   - 完成度受体力消耗、质量影响和运气共同决定

3. **结算奖励**：
   - 成功：根据产品评级获得晶核，可选择临时升级继续挑战
   - 失败：获得少量晶核，重新开始

### 🏆 产品等级
- **C级**：独立作品、外包产品（如博客空间、下载工具）
- **B级**：有市场反响的小产品（如淘宝网、优酷视频）
- **A级**：风靡全国的产品（如微信、支付宝）
- **S级**：世界级产品（如TikTok、王者荣耀）

### 💎 晶核系统
- 完成项目获得晶核奖励（评级越高奖励越多）
- 即使失败也会获得少量安慰晶核
- 使用晶核永久提升眼界、运气、体力上限
- 每个属性可升级10级

### ⚡ 临时升级（9种）
- 灵光一现：眼界+15
- 好运连连：运气+15
- 精力充沛：体力上限+30
- 休息恢复：立即恢复50体力
- 高效开发：任务体力消耗-20%
- 品质大师：任务质量影响+30%
- 快速学习：进度速度+20%
- 冒险家：Boss战成功率+10%，失败不扣晶核
- 完美主义：评级提升一个小等级

## 游戏体验亮点

### 🎨 复古像素风格
- CRT显示器效果（扫描线动画）
- 像素字体和像素边框
- 2000年代怀旧色调
- 命令行风格的界面

### ✨ 爽点设计
- **Todo-list划掉动画**：任务进行时，代办事项随着体力下降逐个完成和划掉
- **实时体力消耗**：可视化的体力条动画，让玩家感受到每个选择的影响
- **市场反响文字**：成功后展示生动的市场评价和产品影响力描述
- **震动反馈**：关键操作时的触觉反馈

### 🎯 策略深度
- 前期选择影响后期：低质量方案会导致Boss战更难
- 体力管理：合理分配体力，避免Boss战前体力不足
- 临时升级选择：不同升级对不同产品类型有不同效果
- 永久升级路线：眼界优先还是体力优先？

## 技术实现

### 技术栈
- **框架**：uni-app (Vue 3)
- **构建工具**：Vite
- **数据存储**：uni.storage (localStorage)
- **平台支持**：H5、微信小程序、APP等全平台

### 项目结构
```
src/
├── data/           # 游戏数据配置
│   ├── products.js    # 30+产品数据
│   ├── solutions.js   # 动态方案生成
│   └── upgrades.js    # 升级配置
├── utils/          # 工具函数
│   ├── storage.js     # 本地存储封装
│   └── gameLogic.js   # 游戏核心逻辑
├── pages/          # 游戏页面
│   ├── home/          # 游戏首页
│   ├── game/          # 游戏主界面
│   ├── quarter-task/  # 季度任务选择
│   ├── task-progress/ # 任务进行页
│   ├── boss-battle/   # Boss战
│   ├── result/        # 项目结算
│   └── upgrade/       # 能力升级
└── App.vue         # 全局样式（复古像素风格）
```

### 核心算法
1. **产品选择算法**：根据眼界值筛选合适等级的产品
2. **方案生成算法**：根据产品类型和季度动态生成不同方案
3. **完成度计算**：综合体力消耗、质量影响、运气值计算
4. **评级算法**：根据完成度和运气调整产品最终评级

## 游戏平衡性

### 数值设计原则
- C级产品：体力需求30左右，适合新手
- B级产品：体力需求55左右，需要一定升级
- A级产品：体力需求85左右，需要充分升级
- S级产品：体力需求115左右，需要多次挑战积累

### 通关路径示例
1. 初始状态挑战2-3个C级产品，获得晶核
2. 升级眼界和体力，挑战B级产品
3. 继续升级，积累临时增益，挑战A级产品
4. 多次A级成功后，体力充足时挑战S级产品
